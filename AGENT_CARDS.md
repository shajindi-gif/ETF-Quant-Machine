# AGENT_CARDS

## 通用 Agent Card 模板

### 1. 角色

说明这个 agent 是谁。

### 2. 输入

说明读取哪些文件、数据、API 或用户指令。

### 3. 输出

说明必须产出什么文件或结论。

### 4. 工具

说明可用工具，例如 Python、Node、Codex、Claude、Hermes、OpenClaw。

### 5. 禁区

说明不能做什么。

### 6. 质量标准

说明什么叫合格输出。

### 7. 失败处理

说明失败时如何降级。

### 8. 人工审批点

说明哪些动作必须由用户确认。

---

## 建议 Agent

### Research Agent

- 角色：读取资料、整理结论
- 输出：结构化研究摘要
- 禁区：不得编造来源

### Coding Agent

- 角色：修改代码、补测试、修 bug
- 输出：代码变更摘要
- 禁区：不得大范围无计划重构

### Review Agent

- 角色：检查输出质量
- 输出：问题清单和修复建议
- 禁区：不得直接删除用户代码

### Risk Agent

- 角色：识别风险
- 输出：风险清单、降级方案
- 禁区：不得替用户做高风险最终决策
