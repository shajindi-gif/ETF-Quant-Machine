# README_FOR_AGENTS

## 项目定位

ETF信号评分与解释机器

## Agent 使用规则

1. 进入本项目后，必须先阅读本文件。
2. 不要删除 .env、.venv、config、data、reports、logs、approvals。
3. 不要在未确认的情况下重命名主目录或移动核心文件。
4. 每次修改前先说明计划，每次修改后说明改动摘要。
5. 每次只改一个明确任务，不要大范围重构。
6. 金融相关项目不得自动下单，不得编造实时行情。
7. 政策/申报相关项目不得给出确定性法律、税务、补贴承诺，必须标记不确定性。
8. 如果数据缺失，必须写明缺失，不得硬编。
9. 修改代码后优先运行 tests 或项目自带检查命令。
10. 所有输出文件优先放入 reports/ 或 logs/。

## 推荐工作流

1. 读取 README.md、README_FOR_AGENTS.md、RUNBOOK.md。
2. 检查项目结构。
3. 给出修改计划。
4. 修改最小必要文件。
5. 运行测试或诊断。
6. 写入 CHANGELOG.md。
7. 输出下一步建议。

## 禁区

- 不删除 .env
- 不删除 .venv
- 不删除 reports
- 不删除 logs
- 不删除用户手写文档
- 不擅自开启后台常驻服务
- 不擅自设置开机启动或定时任务

## 人工审批点

以下动作必须由用户确认：

- 实盘交易
- 自动下单
- 删除文件
- 移动项目目录
- 重置 Git 历史
- 覆盖 .env
- 推送到远程仓库
- 启动长期后台任务
