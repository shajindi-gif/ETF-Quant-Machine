# ACCEPTANCE_CRITERIA

## 通用验收标准

一次修改合格，必须满足：

1. 目标明确。
2. 改动范围可解释。
3. 不破坏原有文件结构。
4. 不删除敏感配置。
5. 运行测试或说明无法测试原因。
6. 输出结果可复查。
7. 关键风险已列出。
8. 下一步动作明确。

## 金融类项目额外标准

1. 不自动下单。
2. 不编造实时行情。
3. 交易建议必须包含验证变量。
4. 仓位建议必须包含风险说明。
5. 报告必须区分事实、推断、假设。
6. 数据缺失必须标记 NA。
7. 不把回测结果当作未来保证。

## 政策产服类项目额外标准

1. 不承诺一定拿到补贴。
2. 不把政策解释当法律意见。
3. 必须列出适用条件和缺口材料。
4. 必须标记政策时效和不确定性。
5. 输出应服务申报路径，而不是泛泛问答。

## 工程类项目额外标准

1. 有 README_FOR_AGENTS。
2. 有 RUNBOOK。
3. 有测试入口或诊断脚本。
4. 有日志目录。
5. 有报告目录。
6. 有 changelog。
